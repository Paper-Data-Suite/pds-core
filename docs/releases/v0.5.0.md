# pds-core v0.5.0

**Release date:** July 14, 2026

**Status:** Supported pre-1.0 Core release

Version 0.5.0 establishes the shared PDS2 routing contract required before
ScoreForm, Quillan, and Concord migrate. It is supported, but it is not called
production-stable and does not promise permanent API stability.

## Architecture summary

PDS2 QR text identifies one expected physical page route with only
`module_id`, `class_id`, `work_id`, and a non-semantic `route_id`. Core resolves
that locator through an immutable persisted registration to a generic
module-owned record reference. Module profiles declare compatibility and
dispatch the retained source page; module-specific identity, evidence, and
processing remain outside Core.

## Compatibility matrix

| Contract | Version |
| --- | --- |
| Core package | 0.5.0 |
| Required Python | >=3.11 |
| Core routing contract | `"1"` |
| QR payload schema | `PDS2` |
| Route-registration schema | `"1"` |
| Routing-failure schema | `"2"` |
| Scan-resolution schema | `"2"` |
| Module profile entry-point group | `paper_data_suite.modules` |

These versions are independent. Downstream packages should use
`pds-core>=0.5,<0.6` and explicitly declare the schema versions they support.

## Breaking changes

- PDS1 and OMR1 parsing, generation, normalization, and fallback support are
  removed. Their payloads are unsupported schemas.
- `QrPayload` and the `pds_core.pds1`, `pds_core.omr1`, and
  `pds_core.qr_payload` runtime modules are removed.
- Universal assignment roots, student-submission directories, and the former
  `pds_core.assignments` routing surface are removed.
- `student_id`, assignment identity, and logical page identity are not shared
  Core routing fields. Modules may retain them behind their own targets.
- Routing-failure and resolution schema version 1 is rejected. Core does not
  provide an automatic conversion.
- Core does not hard-code ScoreForm, Quillan, or Concord routing. Unknown
  module IDs fail until an application registers or discovers a compatible
  module profile.

## New public APIs

- `pds_core.routing_models`: generic work, locator, target-reference,
  registration, and runtime-resolution models.
- `pds_core.route_ids`: secure non-semantic route-ID generation.
- `pds_core.pds2`: strict parser, canonical serializer, and payload limits.
- `pds_core.routes`: module-qualified work, registration, and safe descendant
  paths.
- `pds_core.route_registrations`: exclusive persistence, strict loading, and
  resolution.
- `pds_core.module_profiles`: profiles, registries, installed entry-point
  discovery, and compatibility constants.
- `pds_core.module_dispatch`: single-page and ordered batch dispatch results.
- `pds_core.scan_retention`: retained-source copying and SHA-256 provenance.
- `pds_core.scan_failure_metadata`: version 2 immutable routing failures and
  dispatch-error conversion.
- `pds_core.scan_resolution_metadata`: version 2 append-only linked resolution
  events.

Public examples import from these modules directly. Version 0.5.0 does not add
broad convenience re-exports from `pds_core.__init__`.

## Migration guidance

Follow the [PDS2 module integration guide](../pds2_module_integration.md). In
summary, downstream modules must replace legacy payload handling, persist a
route registration before rendering a PDS2 code, move work beneath the
module-qualified root, expose a compatible `ModuleProfile`, and keep semantic
records and output paths module-owned.

Existing PDS1 workspace data and version 1 failure/resolution files are not
converted by this release. Preserve or migrate them only through a separately
designed, explicit downstream process.

## Downstream module status

This Core release is the prerequisite for, not the completion of, the existing
downstream migrations:

- ScoreForm: `Paper-Data-Suite/pds-scoreform#137`;
- Quillan: `Paper-Data-Suite/pds-quillan#329`; and
- Concord: `Paper-Data-Suite/pds-concord#17`.

No downstream repository or module profile is changed by this release.

## Installation

Install the release wheel attached to the GitHub Release:

```powershell
python -m pip install .\pds_core-0.5.0-py3-none-any.whl
python -m pip check
pds-core --help
```

For a downstream package, declare `pds-core>=0.5,<0.6`. Python 3.11 or newer is
required. PyPI publication is outside this release; use the verified GitHub
Release artifact unless another authorized distribution channel is documented.

## Boundaries and non-goals

This release does not decode or generate QR images, split PDFs, infer semantic
identity, mutate route registrations, add new dispatch behavior, create
module-owned targets, migrate downstream modules, convert historical data, add
runtime dependencies, publish to PyPI, or begin v0.6.0 work.

PDS Core is pre-1.0. Unless otherwise documented, only the latest supported
pre-1.0 minor line receives fixes.

Never post real student data, real rosters or scans, sensitive school or
district information, credentials, or private configuration in a public issue.
